**Step 3 — Extend loadInitialData to read prefill when !isEdit:**

In the `loadInitialData` function, inside the existing `try` block, AFTER `setProveedores(provData)` and INSIDE the `if (isEdit) { ... }` block, add an `else` branch:

```typescript
if (isEdit) {
  // ... existing code to load OC from Firestore ...
} else {
  // Check for pre-populated data from navigation state (e.g., from Generar OC flow)
  const prefill = (location.state as { prefill?: {
    proveedorId?: string;
    proveedorNombre?: string;
    items?: ItemOC[];
  } } | null)?.prefill;

  if (prefill) {
    if (prefill.proveedorId) {
      setProveedorId(prefill.proveedorId);
      // Use proveedorNombre from prefill if available; otherwise try to find from loaded proveedores
      const prov = provData.find(p => p.id === prefill.proveedorId);
      setProveedorNombre(prov?.nombre ?? prefill.proveedorNombre ?? '');
    }
    if (prefill.items && prefill.items.length > 0) {
      setItems(prefill.items);
    }
  }
}
```

**Step 4 — Add location to the useEffect dependency (if needed):**
The existing `useEffect(() => { loadInitialData(); }, [id])` should work as-is since location is read at call time. Do NOT add `location` to the deps array (it would cause re-runs on every navigation). The effect runs once on mount, which is the correct behavior.

**That's all.** Do NOT change any other functionality — form submission, item editing, validation, navigation back. The prefill is purely an initialization concern.

**Edge cases handled by the code above:**
- No prefill state (normal new OC): `prefill` is undefined/null, else branch does nothing, form is empty
- Prefill with no provider: `proveedorId` not set, only items pre-loaded
- Prefill with provider not in proveedores list: falls back to `prefill.proveedorNombre`
