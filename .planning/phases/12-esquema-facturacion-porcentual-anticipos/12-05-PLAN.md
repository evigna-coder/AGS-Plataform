    A) Add private helper near other private methods in `presupuestosService.ts`:

    ```typescript
    /**
     * Phase 12 BILL-02: recompute cuota estados based on current ppto + linked OTs + solicitudes,
     * persist if changed. Idempotent (no-op when nothing changed).
     *
     * MUST be called only AFTER any write to ppto/OT/solicitud has settled — never inside another tx.
     * The atomic single-cuota patch in generarAvisoFacturacion (plan 12-03) handles its own race.
     * Pitfall 2: not for use inside cross-doc tx.
     */
    private async _recomputeAndPersistEsquema(presupuestoId: string): Promise<void> {
      const pres = await this.getById(presupuestoId);
      if (!pres) return;
      if ((pres.esquemaFacturacion?.length ?? 0) === 0) return; // legacy — nothing to do

      // Lazy imports to break circular deps (existing pattern in this service)
      const [{ ordenesTrabajoService }, { facturacionService }] = await Promise.all([
        import('./otService'),
        import('./facturacionService'),
      ]);
      const { recomputeCuotaEstados, cuotasEqual } = await import('../utils/cuotasFacturacion');

      // W4 fix: scoped Firestore query — array-contains on budgets — instead of getAll().
      // pres.numero is the budget identifier (string) used by WorkOrder.budgets[]; verified at
      // services/otService.ts and the data model (see 12-RESEARCH).
      // Composite index requirement: collection 'ordenesTrabajo', single-field auto-indexed on
      // budgets (array-contains is supported on single fields without composite index). Confirm
      // by running once and checking Firestore console for an index suggestion.
      const otsForPres = await ordenesTrabajoService.queryByBudget(pres.numero);
      const solicitudes = await facturacionService.getByPresupuesto(presupuestoId);

      const recomputed = recomputeCuotaEstados(
        {
          estado: pres.estado,
          ordenesCompraIds: pres.ordenesCompraIds,
          preEmbarque: pres.preEmbarque,
          esquemaFacturacion: pres.esquemaFacturacion,
        },
        otsForPres.map(o => ({ otNumber: o.otNumber, estadoAdmin: o.estadoAdmin, budgets: o.budgets })),
        solicitudes.map(s => ({ id: s.id, cuotaId: s.cuotaId ?? null, estado: s.estado })),
      );

      // W2 fix: structural-equality check via cuotasEqual (from plan 12-01).
      // JSON.stringify is non-deterministic after Firestore round-trip on Records — would write churn.
      if (cuotasEqual(recomputed, pres.esquemaFacturacion ?? [])) return;

      const presRef = doc(db, 'presupuestos', presupuestoId);
      await updateDoc(presRef, deepCleanForFirestore({
        esquemaFacturacion: recomputed,
        ...getUpdateTrace(),  // null actor — sync side-effect, not user action
      }) as any);
    }
    ```

    B) **W3 fix — wire recompute into ALL THREE branches of `update(presupuestoId, partial)`**:

    `presupuestosService.update()` (around line 294) is NOT a single updateDoc — it has FLOW-01 / FLOW-03 short-circuit branches:
    - **FLOW-01**: borrador → enviado transition delegates to `markEnviado(...)` and `return;` early.
    - **FLOW-03**: borrador → aceptado transition delegates to `aceptarConRequerimientos(...)` and `return;` early.
    - **Normal path**: tail `updateDoc(...)` for any other partial update.

    If recompute is wired ONLY on the tail path, the borrador→enviado and borrador→aceptado transitions — the very transitions that flip `cuota.estado` for hito='ppto_aceptado' (BILL-02) — will silently skip recompute. Add a shared helper closure and call it on ALL three branches.

    Define near the top of `update()`:

    ```typescript
    const fieldsThatTriggerRecompute = ['estado', 'ordenesCompraIds', 'preEmbarque', 'esquemaFacturacion'];
    const triggers = Object.keys(partial as Record<string, unknown>);
    const shouldRecompute = triggers.some(k => fieldsThatTriggerRecompute.includes(k))
      // Avoid infinite loop: caller-provided authoritative esquema does not need a recompute round-trip.
      && !('esquemaFacturacion' in partial);

    const runRecompute = async () => {
      if (!shouldRecompute) return;
      try {
        await this._recomputeAndPersistEsquema(presupuestoId);
      } catch (err) {
        console.warn('[update.recomputeEsquema] non-blocking error:', err);
      }
    };
    ```

    Then call `await runRecompute();` immediately before EACH `return;` in the function body:

    ```typescript
    // FLOW-01 — borrador → enviado short-circuit
    if (/* FLOW-01 condition */) {
      await this.markEnviado(presupuestoId, /* ... */);
      await runRecompute();   // W3
      return;
    }

    // FLOW-03 — borrador → aceptado short-circuit
    if (/* FLOW-03 condition */) {
      await this.aceptarConRequerimientos(presupuestoId, /* ... */);
      await runRecompute();   // W3
      return;
    }

    // ... normal updateDoc ...
    await updateDoc(presRef, deepCleanForFirestore({ ...partial, ...getUpdateTrace(actor) }) as any);
    await runRecompute();    // W3
    ```

    Document explicitly in code comments: "**Recompute MUST be invoked on all 3 branches**; missing any one breaks BILL-02 because borrador→enviado and borrador→aceptado are the very transitions that flip cuota.estado for hito='ppto_aceptado'."

    **Alternative** (cleaner if FLOW-01/03 helpers can be modified): wrap the recompute call inside `markEnviado` and `aceptarConRequerimientos` themselves (call `_recomputeAndPersistEsquema` at the end of each helper). Pick whichever requires the smallest diff while preserving correctness — but DO NOT rely on a single tail-path call.
