async reservar(input) {
  // Step 1: PRE-FETCH stable data OUTSIDE the tx
  const articulo = await getById(...);
  const candidatas = await unidadesService.getByUbicacion(...);

  // Step 2: pre-generate deterministic IDs
  const movId = crypto.randomUUID();
  const nuevasDestinoIds = Array.from({length: N}, () => crypto.randomUUID());

  // Step 3: runTransaction — READ first, then WRITE
  await runTransaction(db, async (tx) => {
    // READ all candidatas inside tx to validate state under lock
    const snaps = await Promise.all(candidatas.map(u => tx.get(docRef('unidades', u.id))));
    for (snap of snaps) { validate exists + estado === 'disponible' }
    // WRITE — bajas + altas + mov

    tx.set(docRef('movimientosStock', movId), movPayload);

  // Step 4: post-tx fire-and-forget audit

}

Step 1 — Update the signature and return type:

```ts
export async function desagregarUnidades(params: {
  articuloOrigenId: string;
  cantidad: number;
  ubicacion: UbicacionStock;
  solicitadoPorNombre: string;
}): Promise<{ movimientoId: string; cantidadDestino: number }> {
```

Step 2 — Argument validation (synchronous, fail-fast):
- If `params.cantidad <= 0 || !Number.isInteger(params.cantidad)` throw `'Cantidad debe ser entero positivo'`.
- If `!params.solicitadoPorNombre` throw `'solicitadoPorNombre requerido para audit'`.

Step 3 — Pre-fetch (OUTSIDE the tx — stable data):
- Fetch origen via fetchArticulo. If null → throw `'Artículo origen no existe'`.
- If `(origen.equivalencias?.length ?? 0) === 0` → throw `'Artículo origen no tiene equivalencia configurada'`.
- Extract `eq = origen.equivalencias[0]`.
- Fetch destino via fetchArticulo(eq.articuloIdDestino). If null → throw `'Artículo destino no existe'`.
- Fetch candidatas — branch on `_firestoreOverride`. In override mode: `_firestoreOverride.getDocs('unidades', [['articuloId','==',params.articuloOrigenId],['estado','==','disponible'],['activo','==',true],['ubicacion.referenciaId','==',params.ubicacion.referenciaId]])`. In prod mode: same as the override but via real `getDocs(query(...))`. Map docs to UnidadStock.
- Sort candidatas by `createdAt` asc (FIFO).
- Take `candidatas.slice(0, params.cantidad)`.
- If `candidatas.length < params.cantidad` → throw ``Stock insuficiente: ${candidatas.length} disponibles, ${params.cantidad} solicitadas``.

Step 4 — Pre-generate IDs (deterministic paths inside tx):
- `const movId = crypto.randomUUID();`
- `const cantidadDestino = Math.round(params.cantidad * eq.factor);` — round to avoid floating-point surprises when factor is decimal (e.g., 5 * 0.1 = 0.5 → 1 or error?). Actually for decimal factors that produce non-integer cantidadDestino, throw upfront: `if (Math.abs((params.cantidad * eq.factor) - Math.round(params.cantidad * eq.factor)) > 1e-9) throw 'Conversión genera cantidad destino no entera...';`. Use `Math.abs((params.cantidad * eq.factor) - Math.round(params.cantidad * eq.factor)) > 1e-9` as the integer-check tolerance.
- `const nuevasDestinoIds = Array.from({length: cantidadDestino}, () => crypto.randomUUID());`

Step 5 — runTransaction (override-aware branching, single code path via tx interface):

The implementation uses a unified `runOp(tx)` body that operates on a `FirestoreTxDouble`-shaped interface. In override mode, call `_firestoreOverride.runTransaction(runOp)` (the staging method defined by plan 13-02). In prod mode, call `runTransaction(db, async (firebaseTx) => runOp(wrapFirebaseTx(firebaseTx)))` where `wrapFirebaseTx` adapts the firebase Transaction shape to the FirestoreTxDouble shape (1:1 method mapping — both have get/update/set/delete).

Inside `runOp(tx)`:
- READ FIRST — `const snaps = await Promise.all(candidatas.map(u => tx.get('unidades', u.id)));`.
- For each snap: if `!snap.exists()` or `snap.data().estado !== 'disponible'` throw `'Unidad ya no disponible (race con otro proceso)'`. **When this throws inside the override-mode runTransaction, the FirestoreDouble staging mechanism discards all buffered writes — STKE-04b passes because no writes were committed.**
- BUILD payloads using `deepCleanForFirestore`.
- WRITE — for each candidata: `tx.update('unidades', u.id, {estado: 'consumido', ...getUpdateTrace(), updatedAt: now.toDate().toISOString()})`.
- For each newId in nuevasDestinoIds: `tx.set('unidades', newId, deepCleanForFirestore({articuloId: destino.id, articuloCodigo: destino.codigo, articuloDescripcion: destino.descripcion, condicion: 'nuevo', estado: 'disponible', ubicacion: params.ubicacion, activo: true, ...getCreateTrace(), createdAt: now.toDate().toISOString(), updatedAt: now.toDate().toISOString()}))`.
- WRITE MovimientoStock (4 STKE-01 destino fields — NO codigo/descripcion denormalization on the mov):
  ```ts
  tx.set('movimientosStock', movId, deepCleanForFirestore({
    id: movId,
    tipo: 'transferencia',
    subtipo: 'conversion',                            // STKE-01 field #1
    unidadId: candidatas[0].id,
    articuloId: params.articuloOrigenId,
    articuloCodigo: origen.codigo,
    articuloDescripcion: origen.descripcion,
    cantidad: params.cantidad,
    articuloDestinoId: destino.id,                    // STKE-01 field #2 (FK only, no codigo/descripcion)
    cantidadDestino,                                  // STKE-01 field #3
    factorConversion: eq.factor,                      // STKE-01 field #4
    origenTipo: params.ubicacion.tipo,
    origenId: params.ubicacion.referenciaId,
    origenNombre: params.ubicacion.referenciaNombre,
    destinoTipo: params.ubicacion.tipo,
    destinoId: params.ubicacion.referenciaId,
    destinoNombre: params.ubicacion.referenciaNombre,
    motivo: `Conversión ${origen.codigo} × ${params.cantidad} → ${destino.codigo} × ${cantidadDestino} (factor ${eq.factor})`,  // destino codigo embedded HERE per 13-01 trim
    creadoPor: params.solicitadoPorNombre,
    createdAt: now.toDate().toISOString(),
  }));
  ```

**DO NOT add** `articuloDestinoCodigo: destino.codigo` or `articuloDestinoDescripcion: destino.descripcion` fields.
