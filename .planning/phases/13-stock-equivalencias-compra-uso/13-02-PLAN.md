**FirestoreDouble shape required by this plan + plan 13-03:**

```ts
export interface FirestoreDouble {
  /** Read documents from a collection with optional simple where clauses. */
  getDocs(
    collection: string,
    whereClauses?: Array<[string, '==', unknown]>,
  ): Promise<{ docs: Array<{ id: string; data(): Record<string, unknown> }>; empty: boolean }>;

  /** Update mutates fixture state. */
  update(collection: string, id: string, patch: Record<string, unknown>): Promise<void>;

  /** Set creates or replaces a doc in the fixture. */
  set(collection: string, id: string, payload: Record<string, unknown>): Promise<void>;

  /** Delete removes a doc from the fixture. */
  delete(collection: string, id: string): Promise<void>;

  /**
   * Transactional staging mode (Phase 13 plan 13-03 requirement):
   * - Buffers all update/set/delete calls executed via `tx.*` shims inside `fn`.
   * - Applies the buffer to the fixture iff `fn` resolves; discards on throw.
   * - The `tx` object passed to `fn` has the same surface as the top-level methods
   *   PLUS `tx.get(collection, id) => Promise<{exists(): boolean; data(): Record<string, unknown>}>`.
   * - This mirrors Firestore's runTransaction contract so 13-03's `desagregarUnidades`
   *   can share the same code path between prod and override modes by branching only at
   *   entry, not inside the operation.
   */
  runTransaction<R>(
    fn: (tx: FirestoreTxDouble) => Promise<R>,
  ): Promise<R>;
}

export interface FirestoreTxDouble {
  get(collection: string, id: string): Promise<{ exists(): boolean; data(): Record<string, unknown> }>;
  update(collection: string, id: string, patch: Record<string, unknown>): void;
  set(collection: string, id: string, payload: Record<string, unknown>): void;
  delete(collection: string, id: string): void;
}
```

The runTransaction implementation in the test fixture: collect all `tx.update / tx.set / tx.delete` calls into a local buffer, run them only if `fn` resolves, drop them on throw. This is the contract STKE-04b (atomic rollback) asserts against.

Plan 13-03 reuses this interface verbatim — do NOT redefine it there.
