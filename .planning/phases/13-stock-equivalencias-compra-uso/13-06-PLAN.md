**Row ordering rule (per CONTEXT visual spec lines 73-79, formalized by checker m4):**
- `mode === 'origen'`: render origen row FIRST (top), then destino row (the calculated potencial). The user is looking at the origen artículo, so the origen row is the "anchor" they came in for.
- `mode === 'destino'`: render destino row FIRST (top), then origen row beneath. The user is looking at the destino artículo; the destino is the "anchor".
- `mode === 'loading'`: render a loading skeleton (subtle italic text), do NOT render rows.
- `mode === 'none'`: render null.

Note from Open Question #4: the CTA always operates on the origen (lado compra). It is visible from both sides of the dual display. The button text and target action are identical.

**Hook `mode` semantics (revised per checker M5):** the hook MUST NOT return `mode === 'none'` while initial loading is in flight. The discovery sequence is:
1. Hook mounts with articulo → sets loading=true, mode='loading'.
2. If articulo.equivalencias?.[0] exists → mode='origen' immediately (no async needed for mode itself; the destinoArticulo fetch is in-flight but mode is already known).
3. Else → kick off findOrigenDeDestino. While in-flight, mode stays 'loading'.
4. When findOrigenDeDestino resolves to truthy → mode='destino'. When resolves to null/falsy → mode='none'.

Parent component (EquivalenciaDualDisplay) MUST check `mode === 'loading'` BEFORE checking `mode === 'none'`. Old code that short-circuits on `if (mode === 'none') return null;` BEFORE the loading check produces the flicker null → loading → destino — the parent never gets to render the loading state because the hook briefly returns 'none' during the in-flight window.

- `error: string | null`
- `refresh(): void` — re-fetches when the user just performed an action (e.g., desagregar)

Mode invariants:
- `mode === 'loading'` IFF `loading === true` AND no equivalencia data has been resolved yet (no equivalencias[0] AND no origenFetched yet).
- `mode === 'none'` ONLY after `loading === false` AND articulo has no equivalencias[0] AND `origenFetched` is null. Never during initial fetch.
- `mode === 'origen'` as soon as articulo.equivalencias?.[0] exists — even if destinoFetched is still in flight (parent can render origen row with `sd = 0` placeholder until it lands).
- `mode === 'destino'` as soon as `origenFetched` is non-null.

```ts
import { useCallback, useEffect, useState } from 'react';
import { articulosService } from '../services/firebaseService';
import { findOrigenDeDestino } from '../services/equivalenciasService';
import type { Articulo } from '@ags/shared';

type Mode = 'origen' | 'destino' | 'loading' | 'none';

interface UseEquivalenciaDualResult {
  mode: Mode;
  origenArticulo: Articulo | null;
  destinoArticulo: Articulo | null;
  factor: number;
  loading: boolean;
  error: string | null;
  refresh: () => void;
}

export function useEquivalenciaDual(articulo: Articulo | null): UseEquivalenciaDualResult {
  const [origenFetched, setOrigenFetched] = useState<Articulo | null>(null);
  const [destinoFetched, setDestinoFetched] = useState<Articulo | null>(null);
  const [loading, setLoading] = useState(false);
  const [discoveryDone, setDiscoveryDone] = useState(false);  // true once we've resolved findOrigenDeDestino once
  const [error, setError] = useState<string | null>(null);
  const [refreshTick, setRefreshTick] = useState(0);

  const refresh = useCallback(() => setRefreshTick(t => t + 1), []);

  useEffect(() => {
    if (!articulo) {
      setOrigenFetched(null);
      setDestinoFetched(null);
      setDiscoveryDone(false);
      return;
    }

    setLoading(true);
    setError(null);
    setDiscoveryDone(false);

    (async () => {
      try {
        const equivalencia = articulo.equivalencias?.[0];
        if (equivalencia) {
          // Mode: origen — fetch the destino article for stock display
          const destino = await articulosService.getById(equivalencia.articuloIdDestino);
          setDestinoFetched(destino);
          setOrigenFetched(null);
        } else {
          // Maybe this articulo is a destino — query reverse
          const origen = await findOrigenDeDestino(articulo.id);
          setOrigenFetched(origen);
          setDestinoFetched(null);
        }
      } catch (e: any) {
        setError(e?.message ?? String(e));
      } finally {
        setLoading(false);
        setDiscoveryDone(true);
      }
    })();
  }, [articulo, refreshTick]);

  // Derive mode + relevant artículos with the M5 fix.
  if (!articulo) {
    return { mode: 'none', origenArticulo: null, destinoArticulo: null, factor: 0, loading, error, refresh };
  }

  const equivalencia = articulo.equivalencias?.[0];

  if (equivalencia) {
    // Mode: origen — known synchronously from articulo, no flicker
    return {
      mode: 'origen',
      origenArticulo: articulo,
      destinoArticulo: destinoFetched,  // may still be null while in-flight; parent handles
      factor: equivalencia.factor,
      loading,
      error,
      refresh,
    };
  }

  if (origenFetched) {
    const eq = origenFetched.equivalencias?.[0];
    return {
      mode: 'destino',
      origenArticulo: origenFetched,
      destinoArticulo: articulo,
      factor: eq?.factor ?? 0,
      loading,
      error,
      refresh,
    };
  }

  // No equivalencia on this articulo AND no origen pointing here yet.
  // If discovery is NOT yet done, we are still in-flight → return 'loading' (M5 fix).
  // If discovery IS done and there is no origen → return 'none' (terminal).
  if (!discoveryDone) {
    return { mode: 'loading', origenArticulo: null, destinoArticulo: null, factor: 0, loading: true, error, refresh };
  }
  return { mode: 'none', origenArticulo: null, destinoArticulo: null, factor: 0, loading: false, error, refresh };
}
```

**M5 change summary:** the previous draft computed mode in the render path without distinguishing "loading-and-no-data-yet" from "terminal-none". A consumer that did `if (mode === 'none') return null;` BEFORE checking `loading` would render null → loading skeleton → destino over the in-flight window. The new `discoveryDone` flag plus the explicit `'loading'` mode return prevents that flicker.

import { useEffect, useState } from 'react';
import { unidadesService } from '../../services/firebaseService';
import { Button } from '../ui/Button';
import { useEquivalenciaDual } from '../../hooks/useEquivalenciaDual';
import type { Articulo } from '@ags/shared';

interface Props {
  articulo: Articulo | null;
  onDesagregarClick: (origenArticulo: Articulo) => void;
  refreshKey?: number;
}

async function countStockDisponible(articuloId: string): Promise<number> {
  const all = await unidadesService.getAll({ articuloId, estado: 'disponible', activoOnly: true });
  return all.length;
}

export function EquivalenciaDualDisplay({ articulo, onDesagregarClick, refreshKey }: Props) {
  const dual = useEquivalenciaDual(articulo);
  const [stockOrigen, setStockOrigen] = useState<number | null>(null);
  const [stockDestino, setStockDestino] = useState<number | null>(null);

  useEffect(() => {
    // Only fetch stock when we actually have resolved articulos to count for.
    if (dual.mode !== 'origen' && dual.mode !== 'destino') return;
    const origenId = dual.origenArticulo?.id;
    const destinoId = dual.destinoArticulo?.id;
    Promise.all([
      origenId ? countStockDisponible(origenId) : Promise.resolve(0),
      destinoId ? countStockDisponible(destinoId) : Promise.resolve(0),
    ]).then(([o, d]) => {
      setStockOrigen(o);
      setStockDestino(d);
    });
  }, [dual.mode, dual.origenArticulo?.id, dual.destinoArticulo?.id, refreshKey]);

  // M5 fix: handle 'loading' BEFORE 'none' to prevent flicker.
  if (dual.mode === 'loading') {
    return <div className="text-xs text-slate-500 italic" data-testid="equivalencia-dual-loading">Cargando equivalencia…</div>;
  }
  if (dual.mode === 'none') return null;
  if (dual.error) return <p className="text-rose-600 text-xs">{dual.error}</p>;

  const origen = dual.origenArticulo!;
  const destino = dual.destinoArticulo;
  const factor = dual.factor;
  const so = stockOrigen ?? 0;
  const sd = stockDestino ?? 0;
  const potencialesDelOrigen = so * factor;
  const equivalentesDelDestino = factor > 0 ? sd / factor : 0;

  const handleCTA = () => onDesagregarClick(origen);

  // Reusable row renderers — declared once, ordered by mode below.
  const origenRow = (
    <div className="flex items-center gap-2 text-xs flex-wrap" data-testid="dual-row-origen">
      <span className="font-mono text-slate-900">+ {so}</span>
      <span className="font-mono text-slate-900">× {origen.codigo}</span>
      <span className="text-slate-500">sin desagregar</span>
      {destino && so > 0 && (
        <span className="text-[10px] text-slate-500 italic">
          (= {potencialesDelOrigen} × {destino.codigo} potenciales)
        </span>
      )}
      {so > 0 && destino && (
        <Button size="sm" variant="primary" onClick={handleCTA} data-testid="desagregar-ahora-cta">
          Desagregar ahora
        </Button>
      )}
    </div>
  );

  const destinoRow = destino && (
    <div className="flex items-center gap-2 text-xs" data-testid="dual-row-destino">
      <span className="font-mono text-slate-900">{destino.codigo}</span>
      <span className="font-mono text-teal-700">× {sd}</span>
      <span className="text-slate-500">disponibles</span>
      {dual.mode === 'destino' && equivalentesDelDestino > 0 && (
        <span className="text-[10px] text-slate-500 italic">
          └─ (≈ {equivalentesDelDestino.toFixed(2)} × {origen.codigo} equivalentes)
        </span>
      )}
    </div>
  );

  return (
    <div
      className="rounded-md border border-teal-200 bg-teal-50/30 px-3 py-2.5 space-y-2"
      data-testid="equivalencia-dual-display"
    >
      <p className="text-[9px] font-mono font-semibold text-teal-700/70 uppercase tracking-widest">
        Equivalencia compra ↔ uso (factor ×{factor})
      </p>

      {/* Row ordering by mode (m4 fix): the row corresponding to the article the user is looking at sits on top */}
      {dual.mode === 'origen' ? (
        <>
          {origenRow}
          {destinoRow}
        </>
      ) : (
        <>
          {destinoRow}
          {origenRow}
        </>
      )}
    </div>
  );
}
