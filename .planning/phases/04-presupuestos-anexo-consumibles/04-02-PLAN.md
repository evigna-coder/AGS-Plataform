<objective>
Crear el servicio Firestore `consumiblesPorModuloService` y la página admin para gestionar el catálogo `consumibles_por_modulo`. Esta es la fuente de datos que el builder de anexos (plan 04-04) lee al generar el PDF.

Purpose: Sin UI para administrar el catálogo, el feature no es operable — el equipo necesita poder declarar "el módulo G7129A lleva tales consumibles" sin tocar la consola de Firebase.

Output:
1. `consumiblesPorModuloService.ts` con CRUD + lookup por `codigoModulo`.
2. Página admin con lista (búsqueda por código/descripción) + form CRUD inline (patrón espejo de `TiposEquipoList`).
3. Ruta registrada en `TabContentManager` (`/presupuestos/consumibles-por-modulo`).
4. Botón "Consumibles por módulo" en la toolbar de `PresupuestosList` junto a "Tipos de equipo" (entry point coherente).
</objective>
