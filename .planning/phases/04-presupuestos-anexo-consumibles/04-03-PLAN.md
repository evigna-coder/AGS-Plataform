<objective>
Conectar el flag `requiereAnexoConsumibles` (definido en plan 04-01) al editor de plantillas de tipo de equipo y al servicio que las persiste:

1. Agregar checkbox "Anexo" en `ServiciosEditor` (columna en la tabla del form de plantillas).
2. Asegurar que `tiposEquipoService.hydrate()` defaultea el flag a `false` para docs viejos (sin migración).
3. Asegurar que `create/update` persisten el flag (sin crearle problemas de `undefined`).
4. Actualizar el seed (`seedPlantillas.ts`) para que las plantillas iniciales sigan compilando — NO marcar ningún servicio como `requiereAnexoConsumibles=true` por default (el usuario decide caso por caso).

Purpose: Sin esto, el flag definido en 04-01 es un type ghost — no hay UI para tildarlo ni persistencia. El plan 04-04 necesita poder leer servicios con el flag para decidir cuándo generar anexos.

Output: Editor de plantillas con columna "Anexo" funcional. Plantillas existentes intactas. Type-check pasa.

NOTA: Este plan corre en paralelo con 04-02 (admin CRUD) — toca archivos completamente disjuntos (tipos-equipo/ vs consumibles-por-modulo/), por lo que no hay conflicto.
</objective>
